## 一問一答
<details>
<summary>AWS Shield Advancedとはどんなサービス？</summary>
大規模DDoS攻撃からアプリケーションを保護するサービス。
</details>

<details>
<summary>AWS Snowball Edgeとは？</summary>a
物理デバイスを使ってオンプレからクラウドへ大容量データを転送するサービス。
</details>

<details>
<summary>DWHサービスとは？</summary>
大量のデータを高速に分析するために最適化されたデータベースサービスのこと。
</details>

<details>
<summary>Application Migration Serviceの方式を二つを答えよ</summary>
エージェント方式・エージェントレス方式。
</details>

<details>
<summary>Application Discovery Agent とは？</summary>
オンプレミスのサーバーにインストールして 移行に必要な情報を自動収集するためのエージェント です。
</details>

<details>
<summary>AWS Budgetsとは？</summary>
コストや使用量の上限（予算）を設定し、そのしきい値を超えそうなときに通知してくれるサービス
</details>

<details>
<summary>Change Data Captureとは？</summary>
データベースの変更（INSERT / UPDATE / DELETE）をリアルタイムで検出して、別の場所へ継続的に同期する仕組み
</details>

<details>
<summary>CloudFormationのRetain ポリシーとは？</summary>
スタックを削除しても リソースを残す（消さない） 設定。
</details>

<details>
<summary>CloudFormationのDependsOn 属性とは？</summary>
定のリソースが他のリソースより先に作成（または削除）されるよう、依存関係を明示的に指定するための設定 です。
</details>

<details>
<summary>CCloudFrontのオリジングループとは？</summary>
複数のオリジン（S3 や ALB など）を 1つのグループにまとめて、フェイルオーバーできる仕組み のことです。
</details>

<details>
<summary>CloudTrail Lakeとは？</summary>
CloudTrail の証跡ログを自動でデータベース化し、SQL で簡単に検索できるサービス。
</details>

## 1問目：API Gateway × Lambda × Aurora

会社は、Amazon EC2 インスタンスを使用して、ウェブサイトを展開してブログサイトを運営しています。EC2 インスタンスは Application Load Balancer (ALB) の背後にあり、Auto Scaling グループで構成されます。ウェブアプリケーションは、すべてのブログコンテンツを Amazon EFS ボリュームに保存します。動画を投稿する機能をブログに追加したところ、以前よりトラフィックが 10 倍に増えました。ピーク時に、ユーザーがサイトにアクセスすると、読み込むまでに待たされることが多数発生します。

問題を解決する最もコスト効率が高くスケーラブルな方法を選択してください。

- A. Amazon EFS を再構成して、最大 I/O パフォーマンスモードを有効にします。
- B. ブログサイトを更新して、ストレージにインスタンスストアボリュームを使用します。サイトのコンテンツは、起動時に Amazon S3 からインスタンスストアボリュームにコピーし、シャットダウン時に Amazon S3 にコピーします。
- C. S3 をオリジンとした Amazon CloudFront ディストリビューションを設定します。動画データを Amazon EFS から Amazon S3 に移行します。
- D. すべてのサイトのコンテンツ用に ALB をオリジンとした Amazon CloudFront ディストリビューションを設定します。

<details>
<summary>答え</summary>
C

- 最大 I/O モードは「大量の小ファイル・並列アクセス」に向いているが、レイテンシは悪化することがある
- インスタンスストアは EC2 が止まると内容が消える
- 動画コンテンツは EFS より S3 の方が適している
- S3 を CloudFront のオリジンにする構成は AWS の推奨アーキテクチャ。

</details>
